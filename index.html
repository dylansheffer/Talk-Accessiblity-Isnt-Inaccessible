Accessibility Isn't Inaccessible!

⚡ Lightning Talk ⚡
The motivation behind this talk is that people believe that a11y is hard, but it actually just requires a different way of thinking.

What is Accessibility?

Making your site usable by everybody regardless of ability.

RealTalk

It is critical we do this right

  • A11y isn't something we can debate about doing like how we can endlessly debate about the latest javascript framework or programming paradigm.
  • This is something that everybody has to take seriously. It is all of our responsibility as developers to ensure that we preserve the fundamental human right of access to information and self expression for everybody that uses it.
  • Remember that all of us are only an accident or a few years of age from needing to use assistive technology, so if you enjoy being able to freely use the web and would want to do that for the rest of your life, prioritize accessibility.

P.O.U.R

Perceivable

Operable

Understandable

Robust

Perceivable

Available to the senses either through the browser or through assistive technologies.

Mainly through vision and hearing. Assistive technologies include screen readers, screen enlargers, audio captions, etc.

Operable

Users can interact with all controls and interactive elements using either the mouse, keyboard, or an assistive device.

Understandable

Content is clear and limits confusion and ambiguity.

Robust

A wide range of technologies and devices can access the content.

Our sites need to be robust enough to be used in various contexts such as screen readers, screen enlargers, switches, etc. An easy to to ensure this is to give many ways to accomplish common tasks like page navigation and search.
